# Security Policy

## Reporting a Vulnerability

Please report suspected vulnerabilities privately by emailing **stefan_broenner@yahoo.com** with:

- A clear description of the issue
- Affected version/commit and environment details
- Reproduction steps or proof-of-concept
- Potential impact

Do **not** open public GitHub issues for security vulnerabilities.

## Response Process

- We will acknowledge receipt within **3 business days**.
- We will investigate, validate, and provide status updates.
- We will coordinate disclosure timing with the reporter when possible.

## Supported Versions

This project is a research repository and does not currently maintain strict LTS branches.
Security fixes, when available, are typically applied to the latest `main` branch.
